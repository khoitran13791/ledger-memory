# Implementation Plan: Basic Tokenizer Adapters

**Branch**: `001-basic-tokenizer` | **Date**: 2026-03-04 | **Spec**: `/specs/001-basic-tokenizer/spec.md`
**Input**: Feature specification from `/specs/001-basic-tokenizer/spec.md`

## Summary

Implement Phase 1 basic tokenizer capability behind the existing `TokenizerPort` by delivering two selectable behaviors: a deterministic estimator (`1 token ~= 4 chars/bytes`, rounded up) and a model-aligned tokenizer adapter backed by tiktoken for one predefined default model family. Extend SDK composition so tokenizer behavior is selected at engine setup time, invalid configurations fail fast, and invalid tokenizer outputs are rejected with explicit tokenizer errors. Preserve existing domain/application contracts and keep tokenizer implementation details in adapter/sdk layers.

## Technical Context

**Language/Version**: TypeScript 5.x (strict), Node.js >=22.0.0
**Primary Dependencies**: `@ledgermind/application`, `@ledgermind/domain`, `@ledgermind/adapters`, Vitest 3.x, ESLint + `eslint-plugin-boundaries`, Turborepo, tiktoken package for model-aligned counting
**Storage**: N/A for tokenizer implementation (consumed by existing in-memory/PostgreSQL flows)
**Testing**: Vitest unit tests (tokenizer adapters + SDK selection/config validation), integration tests for tokenizer-dependent use cases (`run-compaction`, `store-artifact`)
**Target Platform**: Node.js library runtime (SDK + adapters)
**Project Type**: TypeScript monorepo library (`packages/adapters`, `packages/sdk`, `packages/application`)
**Performance Goals**: deterministic outputs for fixed inputs, exact-match model-aligned token counts versus default-family reference (0% tolerance), no behavioral regression in token-budget-dependent flows
**Constraints**: preserve `TokenizerPort` signature, enforce Clean Architecture dependency rule, reject unsupported tokenizer config at initialization, reject invalid tokenizer outputs with explicit tokenizer error and stop operation, no changes to compaction policy/DAG/persistence schema
**Scale/Scope**: implement + wire two tokenizer behaviors for Phase 1 and validate substitutability across existing core flows
