# Risk Register: PostgreSQL Adapter for Phase 1 Core Engine

| Risk | Category | Likelihood | Impact | Mitigation | Detection | Owner |
|---|---|---|---|---|---|---|
| Idempotency key is not persisted in append path, causing duplicate logical writes or missing conflict detection | Correctness | Medium | High | Align append write path with FR-004 idempotency semantics; add explicit same-key/same-payload and same-key/different-payload tests | `pg-ledger-store` tests + regression cases for idempotency conflicts | Persistence engineer |
| No bounded retry for transient PostgreSQL failures causes avoidable failures and weak recovery behavior | Reliability | Medium | High | Implement deterministic bounded retry in transaction boundary with rollback-before-retry | Transaction-focused integration tests with injected transient failures | Persistence engineer |
| Retryable vs non-retryable failures are not explicitly typed, creating inconsistent caller behavior | Maintainability / Reliability | Medium | High | Extend PostgreSQL error classification contract and tests for retryability surface | Error-mapping unit tests + conformance assertions on failure types | Infrastructure maintainer |
| Concurrent context replacements could corrupt projection if version handling regresses | Concurrency | Medium | High | Preserve strict `expectedVersion` compare-and-swap behavior; keep stale mismatch as typed failure | `pg-context-projection` concurrency tests and contiguous-position integrity checks | Persistence engineer |
| Transaction rollback gaps may expose partial writes across ledger/context/dag/artifact stores | Data integrity | Low-Medium | Critical | Keep all multi-record mutations under `UnitOfWorkPort.execute` and validate rollback scenarios | `pg-unit-of-work` rollback tests + integrity report after fault injections | Persistence engineer |
| DAG lineage corruption (cycles/orphans/coverage violations) may break `expandToMessages` and retrieval trust | Correctness | Low-Medium | High | Retain all 8 integrity checks and block invalid edge creation paths | `checkIntegrity` validation in summary DAG tests + golden/regression suites | Compaction/DAG owner |
| Recursive retrieval/search performance degradation at high history depth may miss SC-004/SC-008 targets | Performance | Medium | High | Validate indexes and query plans; benchmark `expandToMessages` and regex/FTS scenarios at declared scale | Performance test runs at up to 10k events and 100 conversations; latency threshold checks | Performance owner |
| Artifact payload handling regressions could leak mutable binary references to callers | Correctness / Safety | Low | Medium | Keep defensive cloning on binary reads and storage-kind/content-shape checks | `pg-artifact-store` tests for clone semantics and storage-kind enforcement | Persistence engineer |
| Migration drift between schema and adapter expectations can break runtime behavior | Operational | Medium | High | Keep migration artifacts as source of truth and validate migrate-up path in CI | Migration status checks + PostgreSQL harness startup validation | Infrastructure maintainer |
| Contract drift between planning artifacts and implementation may introduce boundary violations | Architecture | Medium | Medium | Keep ports/contracts/research/data-model synchronized and re-run architecture gate before implementation closure | Architecture review checklist + boundary lint/typecheck gates | Tech lead |
